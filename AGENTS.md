# Project Agent Instructions

## 前端设计参考

只要任务涉及新增、重构或优化前端页面，在开始设计与实现前，必须查阅并参考以下站点：

- Dashboard 视觉与数据呈现：https://dribbble.com/search/dashboard
- 高质量网页布局与视觉节奏：https://land-book.com/
- 成熟产品页面、流程与交互模式：https://mobbin.com/
- SaaS 产品界面与信息架构：https://saasinterface.com/

实施时必须做到：

1. 从参考站点提炼与当前任务相关的布局、信息密度、组件层级、按钮、表格、状态反馈和响应式交互方式。
2. 参考结果必须结合本项目的风控业务、现有配色、React + TypeScript 技术栈和既有组件体系落地，不能机械照搬。
3. 保证可访问性、文字清晰度、键盘与触控可用性，不得为了视觉效果牺牲业务效率。
4. 若某个站点需要登录或暂时无法访问，应继续查阅其余可访问站点，并在交付说明中注明实际采用的参考方向。
5. 最终交付时简要说明这些参考如何影响了页面设计。
