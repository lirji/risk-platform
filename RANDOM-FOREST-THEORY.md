# 随机森林理论与反欺诈应用

本平台反欺诈链路的模型分由 **Spark MLlib 随机森林**产出（训练见
`fraud-model-train/.../FraudModelTrainer.java`，线上推理见 `fraud-engine/.../ModelScorer.java`）。
本文从决策树讲到随机森林，说明反欺诈规则如何在森林里被"学习"出来，并列出可落地的拓展点。
工程现状见 [CAPABILITIES.md](CAPABILITIES.md)，规划背景见 [PLAN.md](PLAN.md)。

- 当前超参：`numTrees=60`、`maxDepth=6`、`seed=42`，6 个特征列
- 特征列：`amount`、`daily_amount`、`daily_count`、`hour`、`cross_bank`、`device_new`
- 产物：训练导出 PMML → 线上 `jpmml-evaluator` 进程内推理，输出欺诈概率 `probability(1) ∈ [0,1]`

---

## 1. 决策树：森林的"一棵树"

### 1.1 是什么

决策树是一棵自顶向下的 if-else 树。每个**内部节点**对某个特征做一次阈值判断
（如 `amount > 50000 ?`），每条分支把样本切成两半，**叶子节点**给出预测
（分类树给类别/概率，回归树给数值）。一条样本从根走到叶，路径上的判断串起来
就是一条人类可读的规则。

```
                  amount > 50000 ?
                 /              \
              是                 否
              /                    \
     device_new = 1 ?          hour < 6 ?
       /        \               /       \
   欺诈 0.92   欺诈 0.30    欺诈 0.41   正常 0.03
```

### 1.2 怎么长出来（训练）

核心是**递归地选一个"最能把好坏样本分开"的切分**。衡量"分得好不好"的指标：

- **基尼不纯度 (Gini)**：`Gini = 1 - Σ pᵢ²`。某节点全是欺诈或全是正常时 Gini=0（最纯）。
  Spark 分类树默认用 Gini。
- **信息熵 (Entropy) / 信息增益**：`Entropy = -Σ pᵢ log pᵢ`，切分后熵下降越多越好。
- **方差 (回归树)**：切分后两边数值的方差和越小越好。

每一步：枚举所有特征的所有候选阈值，选使「子节点不纯度加权和」下降最多的那个切分，
直到达到停止条件（`maxDepth`、叶子最小样本数、不纯度增益过小）。

### 1.3 优点与致命缺点

| 优点 | 缺点 |
|---|---|
| 可解释，路径即规则 | **高方差**：训练数据轻微变化 → 树结构剧变 |
| 无需特征归一化 | 容易**过拟合**（长得太深就背样本） |
| 天然处理特征交互（amount×device_new） | 单棵树准确率有限，决策边界是"锯齿状"硬切 |

单棵树的"高方差、易过拟合"正是随机森林要解决的问题。

## 2. 从一棵树到一片森林

### 2.1 核心思想：Bagging + 随机性 → 降方差

随机森林 = **很多棵决策树投票/平均**。它用两层随机性让每棵树"不一样"，
再把它们的预测平均掉，从而抵消单棵树的高方差（统计上：N 个弱相关估计量取平均，
方差约降到 1/N 量级）。

1. **Bagging（行采样 / Bootstrap）**：从训练集有放回抽样，为每棵树造一份略不同的数据。
   本项目 5000 条样本，每棵树看到的是其中一份有放回抽样。
2. **特征随机（列采样）**：每个节点分裂时，只从随机抽到的一部分特征里选最优切分
   （分类任务常用 `√特征数`）。这让树之间**去相关**——否则强特征会让所有树长得很像，
   平均了也降不了多少方差。

```
        训练集 (5000 条标注交易)
   ┌────────┬────────┬───── … ─────┐
   ▼        ▼        ▼              ▼
 采样1     采样2     采样3   …    采样60     ← Bootstrap 行采样
   │        │        │              │
 树1       树2       树3    …      树60      ← 每个节点再列采样选特征
   │        │        │              │
   └────────┴────────┴───── … ─────┘
                  ▼
      分类：60 棵树概率平均 → probability(1)
```

### 2.2 集成为什么更强

- **偏差几乎不变，方差大幅下降**：每棵树仍是低偏差（长得够深能拟合信号），
  平均后方差被压下去，泛化更好。
- **抗噪声、抗过拟合**：个别树被异常样本带偏，会被多数票稀释。
- **代价**：丧失单棵树的可解释性——森林是 60 棵树的平均，不再是一条规则。
  可解释性靠**特征重要度**和 **SHAP**（见 §5）补回来。

### 2.3 关键超参（对应本项目）

| 超参 | 本项目值 | 作用 | 调大/调小 |
|---|---|---|---|
| `numTrees` | 60 | 树的数量 | 越多越稳但越慢，边际收益递减 |
| `maxDepth` | 6 | 单棵树最大深度 | 越深拟合越强但易过拟合 |
| `featureSubsetStrategy` | auto(`√n`) | 每节点候选特征数 | 越小树越去相关、方差越低 |
| `subsamplingRate` | 1.0 | 每棵树行采样比例 | 越小越随机 |
| `minInstancesPerNode` | 1 | 叶子最小样本 | 调大可抗过拟合 |
| `seed` | 42 | 随机种子 | 固定以保证可复现 |

> 经验法则：`numTrees` 先调到收益平台期（看 OOB/验证 AUC 不再升），
> 再用 `maxDepth`、`minInstancesPerNode` 控过拟合。

### 2.4 随机森林 vs GBDT（一句话）

- **随机森林**：树**并行**、相互独立、靠平均**降方差**，抗过拟合、好调、对脏数据稳。
- **GBDT/XGBoost/LightGBM**：树**串行**，每棵拟合上一棵的残差、靠**降偏差**逼近，
  精度通常更高但更易过拟合、更难调。反欺诈工业界两者都常见，本项目先用 RF 求稳，
  可作为后续升级方向（见 §6）。

## 3. 反欺诈规则如何"长"进随机森林

这是本项目最值得理解的一点：**专家规则和模型不是对立的，规则是模型从数据里学到的切分的"显式版本"。**

### 3.1 训练数据里埋了哪些"规则信号"

`FraudModelTrainer.synthesize()` 用一个 logistic 公式合成标签，把领域知识写成了"欺诈倾向"：

```java
double z = -3.0
        + 0.00002  * amount         // 单笔越大越可疑
        + 0.000015 * dailyAmount     // 当日累计越大越可疑
        + 1.2      * deviceNew        // 新设备权重最高
        + 0.6      * crossBank        // 跨行转账加权
        + (hour < 6 ? 1.0 : 0.0)     // 凌晨交易加权
        + 0.05     * dailyCount;      // 当日笔数越多越可疑
double p = 1.0 / (1.0 + Math.exp(-z));   // → 欺诈概率
```

这正是反欺诈领域的常见专家规则：**大额、当日累计高、新设备、跨行、凌晨、高频**。
真实生产中这一步会换成 Hive 宽表里的真实案件标签（见 PLAN §4.2），但信号同构。

### 3.2 森林如何把这些信号变成切分

训练时，每个节点都在找"最能分开欺诈/正常"的切分。因为 `device_new` 对标签影响最大
（系数 1.2），它会带来最大的不纯度下降，于是**大量树会在靠近根的位置切 `device_new=1`**；
`amount`、`hour<6`、`cross_bank` 紧随其后成为次级切分。最终森林里隐含着一批等价规则：

```
路径 device_new=1 ∧ amount>5万 ∧ hour<6  →  叶子欺诈概率 ≈ 0.9   （高危组合）
路径 device_new=0 ∧ amount<1万 ∧ 跨行=0   →  叶子欺诈概率 ≈ 0.02  （低危）
```

模型把单条规则**自动组合成成百上千条带概率的细化规则**，并且阈值（5 万、凌晨 6 点）
是数据驱动学出来的，而非人工拍定。这就是"规则在随机森林中的体现"：
**规则 = 模型的一条根到叶路径；森林 = 海量规则的加权投票。**

### 3.3 特征重要度：反推森林学到的规则权重

随机森林可输出**特征重要度**（每个特征在所有树里带来的不纯度下降之和，归一化）。
对本项目，预期重要度排序大致 `device_new > amount > hour > cross_bank > daily_amount > daily_count`，
与 §3.1 系数同序——这正是验证"模型学对了规则"的手段。

### 3.4 本项目里规则与模型是"双轨并行"，不是二选一

线上决策不是只靠森林。`fraud-engine` 同时跑 **Drools 规则引擎** 和 **随机森林模型分**，
再做决策聚合（取最高 `RiskLevel`：LOW/MEDIUM/HIGH/REJECT）：

```
交易 → ┌─ Drools 规则引擎（硬规则：黑名单、单笔上限、命中即 REJECT）
        ├─ 随机森林模型分（软信号：probability(1) → 概率阈值映射等级）
        └─ 决策聚合（escalate 取最严等级）→ 放行 / 加验 / 复核 / 拒绝
```

**为什么要双轨**：

| 维度 | 规则引擎 (Drools) | 随机森林模型 |
|---|---|---|
| 可解释 | 强，命中哪条一目了然 | 弱，靠重要度/SHAP 解释 |
| 时效 | 改规则即时生效（热发布） | 需重训+重导 PMML |
| 覆盖 | 已知欺诈模式 | 未知/组合模式、灰色地带 |
| 监管/合规 | 易审计 | 需可解释性配套 |
| 风险 | 漏报新型欺诈 | 误杀、概念漂移 |

规则负责"红线 + 可解释 + 即时止血"，模型负责"灰度排序 + 捕捉规则没覆盖的组合"。
模型加载失败时 `ModelScorer` 降级返回 0 分，链路**退化为纯规则**仍可用（PLAN §3.6）。

## 4. 从训练到线上：本项目的完整闭环

```
①合成/Hive标注数据 → ②VectorAssembler 组特征向量 → ③RandomForestClassifier 训练
        → ④测试集 AUC 评估 → ⑤PMMLBuilder 导出 fraud-rf.pmml
        → ⑥fraud-engine 启动加载(jpmml) + verify() → ⑦线上 score() 输出概率
        → ⑧DS 定时重训调度，新 PMML 替换热加载
```

**两个工程要点**：

- **training-serving 对齐**：训练特征列与线上 `ModelScorer.score()` 组装的字段必须一一对应、
  量纲一致。代码里特意把金额 `分 → 元`（`txn.getAmount()/100.0`）对齐训练用的"元"。
  这是模型上线最常见的坑——特征错位会让线上分完全失真。
- **AUC 评估**：`BinaryClassificationEvaluator` 用 `areaUnderROC` 衡量排序能力（区分欺诈/正常
  的能力），对正负样本极不均衡的反欺诈场景，AUC 比准确率更可靠（见 §5 补充 PR-AUC）。

## 5. 评估与可解释性（生产必备）

### 5.1 不均衡场景下选对指标

欺诈占比通常 < 1%，"全判正常"准确率也能 99%，所以**准确率没意义**。该看：

- **AUC-ROC**：整体排序能力（本项目已用）。
- **PR-AUC（查准-查全曲线下面积）**：极度不均衡下比 ROC 更敏感，**建议补上**。
- **召回@固定误报率**：业务真正关心"在可接受的误打扰率下能抓住多少欺诈"。
- **KS 值**：好坏样本累计分布最大差，风控常用。
- **混淆矩阵 + 阈值**：模型出的是概率，要选阈值映射到 LOW/MEDIUM/HIGH/REJECT，
  阈值由"误杀成本 vs 漏报成本"决定，而非默认 0.5。

### 5.2 把森林"解释清楚"

- **特征重要度**：全局看哪些特征驱动决策（§3.3）。
- **SHAP 值**：单笔交易级解释——"这笔为什么被判高危：device_new 贡献 +0.4、amount +0.2"，
  可直接展示给审核员/监管，是反欺诈可解释性的主流方案。
- **OOB（袋外）误差**：Bagging 天然留出未被某树采样的样本，可免额外验证集估泛化误差。

## 6. 拓展点（按性价比排序）

1. **补 PR-AUC / KS / 召回@误报率评估**，并把概率阈值→`RiskLevel` 的映射做成可配置。
   *最低成本、最高收益，先做。*
2. **输出特征重要度 + 接入 SHAP**：训练阶段打印重要度，线上对高风险单返回 Top 贡献特征，
   补齐模型可解释性，过审计/监管。
3. **类别不均衡处理**：`RandomForestClassifier` 设样本权重 / 过采样(SMOTE) / 欠采样，
   或对欺诈类加权，缓解真实数据里欺诈样本稀少。
4. **超参搜索 + 交叉验证**：用 Spark `CrossValidator` + `ParamGridBuilder` 网格搜
   `numTrees`/`maxDepth`/`featureSubsetStrategy`，替代当前手工固定值。
5. **特征扩展**：当前仅 6 维。可加 *速度类*（近 1h/24h 金额与笔数）、*图特征*
   （对手方关联度、团伙）、*序列/行为*（与历史均值偏离）、*设备/IP 指纹*。
   接 Flink 实时特征（`profiling-realtime-flink`）做时间窗聚合。
6. **概念漂移监控 + 自动重训**：欺诈手法会变，模型会衰减。监控线上分分布/AUC，
   触发 DolphinScheduler 重训（项目已有调度雏形），实现规则与模型的持续更新。
7. **升级到 GBDT / XGBoost / LightGBM**：在保住 RF 作 baseline 的前提下，
   用梯度提升追精度，PMML 同样可导出，线上推理链路不变。
8. **模型 + 规则的协同**：用模型重要度/SHAP **反哺规则发现**（模型发现的高频高危路径
   沉淀成 Drools 显式规则，即时生效又可审计），形成"模型探路、规则固化"的闭环。
9. **影子模式 / A-B 上线**：新模型先旁路打分不决策，比对线上效果再切流，降低上线风险。

## 7. 一页速记

- **决策树** = 一串 if-else，按基尼/信息增益贪心切分；优点可解释，缺点高方差易过拟合。
- **随机森林** = Bagging(行采样) + 特征随机(列采样) 造很多去相关的树，平均**降方差**；
  牺牲单树可解释性，换泛化与稳健。
- **反欺诈规则在森林里** = 每条"根→叶"路径就是一条带概率的细化规则；
  特征重要度/SHAP 把森林学到的规则权重反推出来。
- **本项目** = 规则引擎(红线+可解释+热发布) 与 随机森林(灰度排序+捕组合) **双轨并行**，
  模型失败降级为纯规则，training-serving 严格对齐，DS 调度重训。
- **下一步** = 先补不均衡评估指标与可解释性，再做特征扩展、漂移监控与 GBDT 升级。
