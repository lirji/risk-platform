# 面试速记卡：引擎时效性设计（P99 ≤ 50ms）

> 一页临场速记。讲法：先抛主线 → 三层展开 → 接住追问 → 用 trade-off 收尾。
> 配套详解见 [REQUEST-FLOW.md](REQUEST-FLOW.md)（链路+引擎内部）、[ARCHITECTURE-COMPARISON.md](ARCHITECTURE-COMPARISON.md)。

## 🎯 主线（开口第一句）

> **关键路径压到最短，路径上只做纯内存计算；慢操作要么提前做、要么挪到路径外、要么宁可降级也不等。SLA = P99 ≤ 50ms。**

## 三层展开

**① 关键路径零 IO**（最硬的点）
- 链路：预取特征 → 进引擎裁决 → 返回
- 所有 IO 在进引擎**前一次取齐**（Redis 一把拉账号特征快照传入）
- 引擎内**零远程调用**：Drools 纯内存匹配 + jpmml **进程内**推理（不走 RPC）
- → 单请求 RT 稳定毫秒级

**② 引擎本身快且可控**
- **agenda-group 按来源只激活相关规则集**（blacklist/threshold/model），不跑全量
- **fireAllRules(MAX_FIRES=1000)** 硬上限，防规则风暴
- **KieSession 每请求 new + dispose**，无状态、线程安全

**③ 慢操作挪出路径 + 宁可降级不等**
- 算特征/落库 → 决策**返回后**才异步发 Kafka，失败只记日志
- **Sentinel**：流控 QPS + 慢调用熔断（RT>50ms 占比高→熔断），降级返回保守决策，不排队拖垮 SLA
- 依赖故障不阻塞：Redis 挂→空快照继续；模型挂→fraudScore=0 退化纯规则

## 接住追问

| 追问 | 答 |
|------|-----|
| QPS 上不去？ | 决策节点**无状态**，加机器线性扩 + LB；瓶颈在 Redis(分片/读副本) 和 KieSession 创建(可上对象池/stateless session) |
| 异步发 Kafka，特征有延迟？ | 对，特征 **T+ε**；用一点特征新鲜度换关键路径低延迟，反欺诈场景划算 |
| 模型为何不独立部署？ | 独立服务多一跳 RPC，毫秒 SLA 下网络往返是大头 → **进程内推理**(PMML+jpmml) |
| 50ms 怎么保证？ | 关键路径零 IO + Sentinel 熔断兜底 + 监控埋 P99，超阈值降级而非阻塞 |

## 🏁 收尾（显深度）

> 保证时效性的本质是**取舍**：牺牲特征强一致(T+ε)、模型部署灵活性(进程内)、极端下的决策精度(降级)，换关键路径的**确定性低延迟**。反欺诈场景"当场出结果"比"结果绝对最优"更重要，所以这些取舍是对的。
